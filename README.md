# Projets RNCP Niveau 6 – Concepteur Développeur en Science des Données

Ce repo regroupe l'ensemble des descriptions et suivi des projets réalisés dans le cadre de la certification RNCP niveau 6 "Concepteur Développeur en Science des données" (Bootcamp Jedha).

## Liste des projets

- **Bloc 01 – Projet Kayak**
  - Objectif : Analyse de données météo et hôtels pour comprendre les facteurs influençant l’attractivité d’un voyage et améliorer les recommandations de destinations et d’hébergements.

- **Bloc 02 – Projet Steam**
  - Objectif : Analyse de données de jeux vidéo pour comprendre les facteurs qui influencent la popularité et les ventes, et améliorer les décisions de mise en avant sur la plateforme Steam.

- **Bloc 02 – Projet Tinder**
  - Objectif : Analyse de données de speed-dating pour comprendre les facteurs qui influencent l’intérêt mutuel entre participants et améliorer les recommandations de matching.

- **Bloc 03 – Projet Conversion rate**
  - Objectif : Analyse de données marketing pour comprendre les facteurs qui influencent le taux de conversion et optimiser les parcours utilisateurs.

- **Bloc 03 – Projet North Face**
  - Objectif : Analyse de données retail pour comprendre les comportements d’achat et améliorer la segmentation client et les recommandations produits.

- **Bloc 03 – Projet Walmart Sales**
  - Objectif : Analyse de données de ventes pour comprendre les effets saisonniers et promotionnels et améliorer la prévision de la demande.

- **Bloc 04 – Projet AT&T**
  - Objectif : Analyse de données textuelles SMS et améliorer les stratégies de de détection des spams.

- **Bloc 05 – Projet Getaround**
  - Objectif : Analyse de données de location de véhicules pour comprendre les facteurs influençant l’usage et optimiser la tarification et les recommandations de véhicules.

## Organisation

Chaque dossier `Bloc_XX_Projet_*` contient (à minima) :

- `README.md` : description du projet (contexte, problématique, méthodologie, résultats, mode d’emploi)
- `data/` : données (échantillons uniquement dans le repo)
- `notebooks/` : analyses exploratoires et modèles
- `src/` : scripts réutilisables (si applicable )
- `Présentation/` : présentation finale (PDF, slides, etc.)
